## Функциональные возможности

- Генерация и проверка лицензий для различных программных продуктов
- Поддержка продуктов JetBrains, GitLab, FinalShell, MobaXterm и JRebel
- RESTful API интерфейс, созданный с использованием фреймворка Gin
- Запланированные задачи с использованием cron
- Хранение данных в базе данных с помощью GORM (поддержка MySQL/SQLite)
- Безопасное шифрование с использованием RSA
- Многоязычная поддержка (упрощенный китайский, традиционный китайский, русский, английский, японский, корейский)

## Требования

- Go 1.24 или выше
- База данных MySQL (или SQLite для разработки)
- Docker (опционально, для контейнеризованного развертывания)

## Установка

### Вариант 1: Прямая установка

1. Клонировать репозиторий
   ```
   git clone https://github.com/nannanStrawberry314/license.git
   cd license
   ```

2. Установить зависимости
   ```
   go mod download
   ```

3. Настроить переменные окружения (скопировать .env.example в .env и изменить по необходимости)

4. Собрать и запустить
   ```
   go build -o license-server
   ./license-server
   ```

### Вариант 2: Развертывание Docker

1. Собрать Docker образ
   ```
   docker build -t license-server .
   ```

2. Запустить с помощью docker-compose
   ```
   docker-compose up -d
   ```

## Конфигурация

Конфигурация осуществляется через переменные окружения и файл `.env`:

- `HTTP_HOST`: Адрес хоста для привязки сервера
- `HTTP_PORT`: Порт для прослушивания
- `DB_TYPE`: Тип базы данных (mysql или sqlite)
- `DB_DSN`: Строка подключения к базе данных

## Конфигурация GitHub Actions

Этот проект включает рабочие процессы GitHub Actions для автоматизированной сборки, тестирования и выпуска. Для использования этих рабочих процессов необходимо настроить следующие секреты репозитория:

- `HUB_USER`: Ваше имя пользователя Docker Hub
- `HUB_PASS`: Ваш пароль Docker Hub
- `HUB_REPO`: Имя репозитория Docker Hub
- `PUBLIC_REPO_TOKEN`: (Опционально) Персональный токен доступа с правами записи в публичный репозиторий
- `PUBLIC_REPO`: (Опционально) Путь к публичному репозиторию в формате `имя_пользователя/репозиторий`

Рабочие процессы включают:
- Сборку и тестирование при каждом пуше
- Сборку и публикацию Docker образа по тегам или ручным триггерам
- Создание GitHub релизов
- Синхронизацию релизов в публичный репозиторий (только для ручных триггеров, требуются `PUBLIC_REPO_TOKEN` и `PUBLIC_REPO`)

## API Endpoints

API предоставляет различные эндпоинты для управления лицензиями:

- `POST /v1/generate`: Генерация новой лицензии
- `POST /v1/validate`: Проверка существующей лицензии
- `GET /v1/status`: Проверка статуса сервиса

Для получения подробных инструкций по использованию обратитесь к документации API.
